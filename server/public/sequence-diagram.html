Diagrama de Sequência — ALYA × Nuvemshop

1. Conexão da Loja (Autenticação por Access Token)

sequenceDiagram actor Lojista participant ALYA as ALYA Frontend participant Backend as ALYA Backend participant NS as API Nuvemshop Lojista->>ALYA: Insere Access Token + Store ID ALYA->>Backend: POST /api/nuvemshop/connect Backend->>NS: GET /v1/{storeId}/store
Authentication: bearer {token} NS-->>Backend: 200 OK · dados da loja Backend->>Backend: Criptografa token (AES-256-GCM)
Salva em nuvemshop_config Backend->>NS: DELETE /webhooks existentes (limpeza) loop Para cada evento Backend->>NS: POST /webhooks
{event, url: /api/nuvemshop/webhook} end NS-->>Backend: 200 OK · IDs dos webhooks registrados Backend-->>ALYA: {success, storeName, webhooksRegistered} ALYA-->>Lojista: Loja conectada com sucesso ✓

2. Sincronização Manual de Pedidos

sequenceDiagram actor Lojista participant ALYA as ALYA Frontend participant Backend as ALYA Backend participant NS as API Nuvemshop participant DB as Banco de Dados Lojista->>ALYA: Clica em "Sincronizar Pedidos" ALYA->>Backend: POST /api/nuvemshop/sync/orders Backend->>DB: Busca token criptografado + last_sync_orders DB-->>Backend: config da loja Backend->>Backend: Descriptografa token loop Paginação (per_page=200, via header Link) Backend->>NS: GET /orders?payment_status=paid&per_page=200 NS-->>Backend: [ lista de pedidos ] end loop Paginação (authorized) Backend->>NS: GET /orders?payment_status=authorized&per_page=200 NS-->>Backend: [ lista de pedidos ] end loop Para cada pedido não sincronizado Backend->>DB: Verifica nuvemshop_sync_map DB-->>Backend: não encontrado Backend->>DB: INSERT transaction (Receita · Venda Online) Backend->>DB: INSERT nuvemshop_sync_map end Backend->>DB: Atualiza last_sync_orders Backend-->>ALYA: {imported, skipped, errors} ALYA-->>Lojista: X pedidos importados ✓

3. Webhook — Pedido Pago em Tempo Real

sequenceDiagram participant NS as Nuvemshop participant Backend as ALYA Backend participant DB as Banco de Dados NS->>Backend: POST /api/nuvemshop/webhook
x-linkedstore-hmac-sha256: {hmac}
Content-Type: application/json alt NUVEMSHOP_CLIENT_SECRET configurado Backend->>Backend: Valida HMAC-SHA256
timingSafeEqual(expected, received) else Sem client secret Backend->>Backend: Log de aviso (sem bloqueio) end Backend->>Backend: Parse payload
storeId = payload.store_id
event = payload.event Backend->>DB: Busca config por store_id DB-->>Backend: config encontrada Backend-->>NS: 200 OK · {received: true}
(resposta em < 3 segundos) Note over Backend: Processamento assíncrono alt event = order/paid Backend->>DB: Verifica nuvemshop_sync_map DB-->>Backend: não encontrado Backend->>DB: INSERT transaction (Receita · Venda Online) Backend->>DB: INSERT nuvemshop_sync_map else event = order/cancelled Backend->>DB: Verifica nuvemshop_sync_map DB-->>Backend: encontrado Backend->>DB: INSERT transaction (Despesa · Estorno Nuvemshop) else event = product/created ou product/updated Backend->>DB: Upsert produto + atualiza estoque else event = customer/created ou customer/updated Backend->>DB: Upsert cliente (email/telefone criptografados) end

4. Sincronização Manual de Produtos e Clientes

sequenceDiagram actor Lojista participant Backend as ALYA Backend participant NS as API Nuvemshop participant DB as Banco de Dados Lojista->>Backend: POST /api/nuvemshop/sync/products loop Paginação (per_page=200, via header Link) Backend->>NS: GET /products?per_page=200&page=N NS-->>Backend: [ lista de produtos ] end loop Para cada produto Backend->>DB: Verifica nuvemshop_sync_map alt Novo produto Backend->>DB: INSERT produto (nome, preço, estoque) Backend->>DB: INSERT nuvemshop_sync_map else Produto existente Backend->>DB: UPDATE produto (preço, estoque) end end Backend-->>Lojista: {imported, updated, errors} Lojista->>Backend: POST /api/nuvemshop/sync/customers loop Paginação (per_page=200, via header Link) Backend->>NS: GET /customers?per_page=200&page=N NS-->>Backend: [ lista de clientes ] end loop Para cada cliente Backend->>Backend: encrypt(email) · encrypt(phone)
AES-256-GCM Backend->>DB: Upsert cliente + nuvemshop_sync_map end Backend-->>Lojista: {imported, updated, errors}

6. Desconexão da Loja

sequenceDiagram actor Lojista participant Backend as ALYA Backend participant NS as API Nuvemshop participant DB as Banco de Dados Lojista->>Backend: DELETE /api/nuvemshop/disconnect Backend->>DB: Busca config + descriptografa token Backend->>NS: GET /webhooks NS-->>Backend: [ lista de webhooks ] loop Para cada webhook Backend->>NS: DELETE /webhooks/{id} NS-->>Backend: 200 OK end Backend->>DB: DELETE nuvemshop_config Backend->>DB: DELETE nuvemshop_sync_map Backend-->>Lojista: {success: true}
